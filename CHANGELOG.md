### Fixed

- **`PUT /business/capacity` rejected every save with a 422** (2026-07-18) — the new Capacity editor (see Added, below) sent all 7 weekday keys on every save, closed days as `slots.<day>: []`. The live backend's `UpdateCapacityRequest` validates a present day as `['array', 'size:2']` — there is no "closed" shape, so an empty array always fails (`"The slots.tue field must contain 2 items."`). Confirmed by reading `osta_backend`'s `UpdateCapacityRequest`/`UpdateCapacityAction` directly: `slots` fully replaces `working_hours` server-side (no per-day merge), so omitting a closed day's key is both valid and correct — it still clears any hours that day previously had. `CapacityScreen._submit()` now only includes open days in the `slots` map. `flutter analyze` clean, full suite 147/147.

  > ‏**`PUT /business/capacity` كان يرفض كل حفظ بخطأ ٤٢٢** (٢٠٢٦-٠٧-١٨) — كانت شاشة السعة الجديدة ترسل مفاتيح الأيام السبعة كلها، والأيام المغلقة كمصفوفة فارغة `[]`. يتحقق الخادم من أن اليوم الموجود يحوي عنصرين بالضبط ولا شكل لـ"مغلق" — فترفض المصفوفة الفارغة دومًا. تأكّد ذلك بقراءة مصدر الخادم مباشرة: `slots` تستبدل `working_hours` بالكامل، فحذف مفتاح اليوم المغلق صحيح ويكفي لمسح ساعاته السابقة. صار `_submit()` يرسل الأيام المفتوحة فقط. التحليل نظيف والاختبارات ١٤٧/١٤٧.

- **Map stuck on the fallback / infinite "locating" spinner** (2026-07-18) — `GeolocatorLocationService.currentPosition()` called `Geolocator.getCurrentPosition()` with **no time limit**, so on emulators and cold GPS (no active fix) it waited forever: the map's `locating` spinner never cleared and the camera stayed on the downtown-Cairo fallback instead of centering on the user. Bounded the fresh-fix wait to 10 s and, on timeout, fall back to `getLastKnownPosition()` so the map still centers on the last cached fix (surfacing an error with retry only when there's no cached fix either). Fixed in the shared service, so all three callers — the customer map, the onboarding pin picker, and Home's nearby rail — benefit. `flutter analyze` clean.

  > ‏**الخريطة عالقة على الموقع الاحتياطي / دوران «تحديد الموقع» بلا نهاية** (2026-07-18) — كانت `getCurrentPosition()` تُستدعى دون حدّ زمني، فعلى المحاكيات وعند برودة الـGPS تنتظر للأبد: لا يتوقّف الدوران ولا تتمركز الكاميرا على المستخدم بل تبقى على وسط القاهرة الاحتياطي. حُدّ الانتظار بعشر ثوانٍ، وعند انتهائه يُستخدم آخر موقع مخزّن `getLastKnownPosition()` حتى تتمركز الخريطة، مع عرض خطأ وإعادة محاولة فقط إن لم يوجد موقع مخزّن. الإصلاح في الخدمة المشتركة، فيستفيد منه المستدعون الثلاثة: خريطة العميل، ومنتقي الدبّوس في الانضمام، وشريط «القريب منك» في الرئيسية. التحليل نظيف.

### Changed

- **Five features restructured to the auth-style feature shape** (2026-07-24) — `customer/garage`, `customer/map`, `customer/booking`, `shared/profile` and `business/onboarding` now match the canonical `shared/auth` layout (branch `refactor/split-features`, one commit per feature, zero behavior change — same routes, same requests, same states). Per feature: models consolidated under `data/models/` (killing the `data/model/` / `data/Model/` drift), every repo got an `abstract interface class` **domain contract** (`domain/<x>_repository.dart`, 8 new: garage, maintenance, centers, center-detail, booking, profile, address, business-onboarding) with the implementation renamed `*RepositoryImpl` in `data/` taking `ApiClient` **via constructor** — the static `GetIt.instance<ApiClient>()` repo style (`BookingRepo.list(...)` et al.) is gone from these features. Presentation split into per-flow sub-folders like auth's `login/`/`register/` (e.g. booking → `my_bookings/` · `create/` · `live/`; map → `map/` · `center_detail/`; profile → `profile/` · `addresses/`; onboarding → `identity/` · `catalog/` · `profile/` · `address/`), and `*Screen` pages renamed `*Page`. Blocs/cubits now take their repo via constructor and come from `getIt` (factories; id-scoped ones — `BookingCreateBloc`, `BookingDetailBloc`, `CenterDetailBloc`, `MaintenanceCubit` — via `registerFactoryParam`, a first for this codebase). `HomeBloc`/`DashboardBloc` were retyped onto the new contracts. Bonus deletion: `business/onboarding`'s `data/Model/servise_model/` (typo'd `ServiseModel`) and `BusinessCatalogRepo` had **zero references** and are removed. `ProfileCubit` lost its `GetIt` fallback (required positional repo) and its test fake now implements the contract instead of extending the impl. `flutter analyze` 0 errors / 22 infos (down from 29 — the dead files took theirs along), full suite **147/147**.

  > ‏**إعادة هيكلة خمس ميزات على نمط ميزة المصادقة** (٢٠٢٦-٠٧-٢٤) — صارت ميزات المرآب والخريطة والحجوزات والملف الشخصي وانضمام الأعمال مطابقة لبنية `shared/auth` المرجعية (فرع واحد، commit لكل ميزة، وبلا أي تغيير سلوكي). لكل ميزة: النماذج تحت `data/models/`، وعقد مجرّد لكل مستودع في `domain/` (٨ عقود جديدة) مع تنفيذ يأخذ `ApiClient` عبر المُنشئ بدل النمط الساكن، وتقسيم العرض إلى مجلدات فرعية لكل تدفق مثل auth، وإعادة تسمية الصفحات من `*Screen` إلى `*Page`. صارت كل الـBloc/Cubit تُحقن مستودعاتها عبر المُنشئ وتُسجَّل في `injection.dart` (والمرتبطة بمعرّف عبر `registerFactoryParam`). حُذفت شيفرة ميتة بلا أي مرجع (`ServiseModel` و`BusinessCatalogRepo`). التحليل ٠ أخطاء / ٢٢ ملاحظة (انخفضت من ٢٩)، والاختبارات ١٤٧/١٤٧.

### Added

- **Platform-adaptive pickers and controls across the app** (2026-07-18) — every date/time picker in the app was hardcoded to the Material dialog, rendering wrong on an iOS-styled screen (reported via a live device screenshot). - **The user's GPS coordinates were about to be logged in release builds** (2026-07-15) — `buildAppDio` added `PrettyDioLogger(responseBody: false)` unconditionally.
