# Changelog

All notable changes to the OSTA app are documented here.
Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/); the project adheres to [Semantic Versioning](https://semver.org/).

## [Unreleased]

### Changed

- **Adopted a `develop`/`main` branching model** (2026-07-06) — introduced a long-lived **`develop`** integration branch: all feature/chore branches are now cut from `develop` and PR into `develop`, while **`main`** becomes the protected release branch, advanced **only** by a `develop → main` release PR + SemVer tag (`v0.<n>.0` per milestone, `v1.0.0` = MVP). Retargeted the open first-run PR ([#67](https://github.com/YoussefSalem582/Osta-App/pull/67)) to `develop`. Updated every workflow doc and AI-agent config (`AGENTS.md`, `CONTRIBUTING.md`, `README.md`, `ARCHITECTURE.md`, `OSTA_plan.md` §13, `OSTA_TODO.md`, `CURRENT_STATUS.md`, `guides/01_folder_structure.md` + `guides/03_how_to_add_new_feature.md`, `reference/COMMON_PITFALLS.md`, `.cursor/rules/git-commits.mdc`, `.claude/commands/{add-feature,new-screen}.md`, and the `.agents`/`.cursor` `add-feature` skills) and added `develop` to the CI `push` triggers (`.github/workflows/ci.yml`).

- **Branch-naming rule tightened across all docs and AI-agent configs** (2026-07-05) — branch names must be hand-written, descriptive, lowercase kebab-case (`<type>/<issue>-<slug>`, e.g. `feat/44-booking-funnel`, `fix/auth-401-loop`); auto-generated/tool-default names (random suffixes, `claude/...`, `cursor/...`, `codex/...`) are forbidden and must be renamed (`git branch -m`) before opening a PR. Updated: `AGENTS.md` (naming table + Git & PRs), `CONTRIBUTING.md`, `README.md`, `ARCHITECTURE.md`, `OSTA_plan.md` (§2 NEVER list + §13.1), `OSTA_TODO.md` (per-epic ritual), `osta_readme_files/guides/03_how_to_add_new_feature.md`, `osta_readme_files/CURRENT_STATUS.md`, `.cursor/rules/git-commits.mdc`, `.claude/commands/add-feature.md` + `new-screen.md`, and the `add-feature` skills in `.agents/` and `.cursor/`.

- **Docs synced to the plain-Dart refactor + made bilingual (EN/AR)** (2026-07-05) — after the "defer advanced tooling" refactor ([PR #69](https://github.com/YoussefSalem582/Osta-App/pull/69)) dropped `fpdart`, `freezed`, `json_serializable`, `injectable`, `build_runner`, build flavors, the `/gallery` route, and the 3-job CI matrix, the whole documentation set was corrected to match (sealed `Failure` + `try`/`catch`; plain `Equatable` models + hand-written JSON; manual `get_it` DI; single `BASE_URL`; one `format · analyze · test` CI job) and cross-linked to [`docs/ROADMAP.md`](docs/ROADMAP.md). ADR 004 (error handling), ADR 005 (codegen), and ADR 008 (CI) rewritten accordingly. `AGENTS.md`, `CLAUDE.md`, and every `osta_readme_files/` doc now carry Arabic (RTL) alongside English prose.

### Added

- **M1 Auth — email/password surfaces + secure storage + password recovery** (epic [#35](https://github.com/YoussefSalem582/Osta-App/issues/35)) (2026-07-06) — extended the shared auth surface toward the M1 email+password epic (no phone OTP): the **register** form now collects a unique username and a required Egyptian **+20** phone (masked prefix, normalized to E.164 on submit), confirms the password, and gates submit behind a **Terms & Privacy** checkbox; the **login** form gains a password **visibility toggle** and a **"forgot password?"** link. Added a two-step **password-recovery flow** — `ForgotPasswordPage` (`POST /forgot-password`) → `ResetPasswordPage` (`POST /reset-password`, email/token prefilled from a reset-link's query params) — driven by a new `PasswordRecoveryCubit`, on new `/auth/forgot-password` + `/auth/reset-password` routes whitelisted in the `resolveRedirect` guard. `AuthRepository` gained `logout` (`POST /auth/logout`, best-effort revoke that always clears the token pair — wired through `SessionController.signOut`), `forgotPassword`, and `resetPassword`. Server **422** responses now surface as inline per-field errors (email/username/phone/password) via `AuthState.fieldErrors`, and validators moved to a shared `AuthValidators` helper. EN + AR strings for the whole surface; endpoints follow the canonical catalogue (`/auth/logout`, public `/forgot-password` + `/reset-password`). Tests cover the new repo calls, the recovery cubit, the validators, and the password toggle.
- **Debug-only login prefill** (2026-07-07) — `AuthPage` prefills the email/password fields with the App Review / QA test account (`test@osta.com` / `osta123123`) under `kDebugMode` only; release builds compile the block out and ship empty fields. Speeds local sign-in and gives App Review a one-tap login. The account must exist backend-side — the app still authenticates against `/auth/login`.
- **First-run flow & 4-role split** (epic [#32](https://github.com/YoussefSalem582/Osta-App/issues/32), [PR #67](https://github.com/YoussefSalem582/Osta-App/pull/67)) (2026-07-06) — OSTA's "one download, then split" first-run model: `SplashPage` bootstraps persisted `{token, activeRole, locale}`; a one-time `LanguagePage` (ar/en, RTL-first); a `RoleChooserPage` where `customer` + `business` are live and `mechanic` + `tow` render disabled ("coming soon"); a pure `resolveRedirect` guard driving a single `go_router` global redirect (splash → language → chooser → auth → shell, no guest path); `account_type = activeRole` on register/login; wrong-shell self-heal from `me.type` + toast; "switch role" (clears role, keeps token) and sign-out; customer + business shells. Router state lives in a `SessionController` cubit (the routing source of truth). **Rebased onto current `main` and adapted to the post-[#69](https://github.com/YoussefSalem582/Osta-App/pull/69) plain-Dart conventions** — the feature's `injectable`/`freezed`/`fpdart` and the dev `/gallery` route were dropped: session/auth deps are now registered by hand in `configureDependencies()` (`AuthCubit` as a factory) and the `AppRouter` is built with the `SessionController` singleton.
- **`OSTA_TODO.md` — zero-to-production checklist** (2026-07-05) — the trackable companion to `OSTA_plan.md`: a phase-by-phase checkbox roadmap from the (already-done ✅) M0 foundation through every milestone (M0 wrap → M1…M5 → Shop/Home/Notifications, with per-epic owners, branches, and key ACs, and a 🏷️ release tag at each phase boundary) to a full **Phase 9 production-readiness gate** new to the docs: app identity/platform config, production credentials (Maps, Firebase/FCM, Google/Apple sign-in, Paymob, Reverb), signing + Play/App Store setup (data safety, privacy labels), release CI, crash-reporting ADR, hardening drills (offline, realtime, push, payments, performance, accessibility, security, l10n), beta tracks, staged `v1.0.0` rollout, and post-launch/Phase-2 items.
- **`OSTA_plan.md` — master AI-agent build instructions** (2026-07-05) — a root-level, English, system-prompt-style execution plan for AI agents continuing the repo. Grounded in the 31 open epics (tracker [#61](https://github.com/YoussefSalem582/Osta-App/issues/61)) and the repo canon: 11 non-negotiable mandates (Clean Architecture + BLoC, dark/light theme, responsive, ar/en RTL-first l10n, animations/transitions, reusable widgets + centralized resources, `talker` logging, `skeletonizer` loading, docs, offline-first, clean git graph); four explicit canon **amendments** (talker replaces `pretty_dio_logger`; skeletonizer everywhere; a new `lib/core/offline/` sqflite cache-then-network + sync-queue spec with per-feature offline policies; a SemVer release/tag convention — `v0.<n>.0` per milestone, `v1.0.0` = MVP); precedence rules (backend contract → `AGENTS.md`/ROADMAP → plan → feature docs → epics, whose stale codegen/Riverpod boilerplate is explicitly overridden); and a per-epic milestone execution plan (M0 finish → M1–M5 → Shop/Home/Notifications → M6/Phase 2) with branches, key ACs, endpoints, offline policy, and package policy per epic.
- **Official Dart & Flutter agent skills vendored** (2026-07-05) — curated copy of the [Flutter](https://github.com/flutter/skills)/[Dart](https://github.com/dart-lang/skills) teams' Agent Skills ([announcement](https://blog.flutter.dev/introducing-skills-for-dart-and-flutter-23837c6ec0ae)) into `.claude/skills/` for Claude Code auto-discovery: 14 skills installed (widget/integration/unit testing, widget previews, responsive layout, layout debugging, hand-written JSON serialization, go_router routing, ARB/gen-l10n localization, coverage, runtime-error debugging, package-conflict resolution, static analysis, pattern matching); 7 excluded for conflicting with OSTA conventions (`http`-package usage vs `ApiClient`, MVVM/`Result` vs BLoC + thrown `Failure`s, `mockito`/`build_runner` and FFI codegen vs no-codegen) — provenance, precedence rules, and update workflow in [`.claude/skills/README.md`](.claude/skills/README.md).
- **AI-agent config set** (2026-07-05) — per-tool instruction files mirroring a proven multi-agent layout: root `CONTRIBUTING.md`, `ARCHITECTURE.md`, `CURSOR.md`; `.agents/` (generic shim `AGENTS.md` + 8 scoped rules + 3 project-tuned skills `add-feature`/`add-api`/`add-language`); `.claude/` (8 slash commands + `settings.json` permission allowlist); `.codex/AGENTS.md`; `.cursor/` (9 `.mdc` rules + skills mirror); `.github/copilot-instructions.md` + `docs.yml` (markdownlint + link-check).
