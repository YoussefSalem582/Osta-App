## [Unreleased]

### Changed

- **Auth form fields UX pass — icons, autofill, focus/error rings, persistent +20** (2026-07-08) — a usability pass on the shared `AppTextField` (`lib/shared/ui/app_text_field.dart`) and the input theme (`lib/core/theme/app_theme.dart`). Fields now render a **brand-green focus ring** and an **error ring** (were flat/borderless with no focus feedback), and leading/trailing icons **tint to the brand colour while focused**. Every auth field gained a contextual **leading icon** (email→mail, password/confirm→lock, first/last name→person, username→`@`, reset code→ticket). Added **OS autofill hints** across the flow (email, given/family name, new username, current/new password, one-time reset code, national phone) with the login/register form wrapped in an **`AutofillGroup`** so password managers can offer + save credentials; name fields capitalise words. Each field now uses a Material **floating label** (`AppTextField` renders the field name as the in-field placeholder in a **muted colour**, which floats to the border in **brand green** once focused/filled) instead of a separate label above the box — so the placeholder reads as the field name and is clearly distinct from full-colour typed text (the earlier example hints and `hint*` strings were dropped, and the theme now sets `hintStyle`/`labelStyle`/`floatingLabelStyle`). The Egyptian **phone field** shows an **always-visible `+20` dial prefix** (with a divider) instead of the old focus-only `prefixText`. The password **eye toggle** uses outlined icons and a localized **Show/Hide password** tooltip (new `showPassword` / `hidePassword` strings, EN + AR). The theme change benefits every text field app-wide. `flutter analyze` clean; 107 tests pass (the eye-toggle component test was updated to the outlined icons).

- **Branded app launcher icon + native splash screen** (2026-07-08) — replaced the default Flutter launcher icon with the OSTA mark (white wordmark + smile on brand green, source `assets/images/app_icon.png`) and added a native OS launch splash matching the in-app `SplashPage` (white OSTA logo on brand green `#0E7A3B`) so the OS splash → first Flutter frame is seamless. Wired via two one-time native-asset generators added as dev deps — `flutter_launcher_icons` (^0.14.3) and `flutter_native_splash` (^2.4.6), configured in `pubspec.yaml`, run manually (not `build_runner`). Android gets per-density mipmaps **plus an adaptive icon** — background `#0E3725`, and a padded foreground (`assets/images/app_icon_foreground.png`, generated by scaling the wordmark into the safe zone) so the circular/rounded launcher mask can't clip it; iOS gets the full `AppIcon.appiconset` (all sizes) + `LaunchImage`/`LaunchScreen`. Android 12+ splash uses the same padded foreground on the brand-green background. No Dart/runtime changes; analyze clean, all 107 tests pass.

- **Auth + language + role screens UI/UX refactor + branded logo on every screen** (2026-07-08) — introduced a shared **`BrandScaffold`** (`lib/shared/ui/brand_scaffold.dart`) — a collapsing brand-green **`SliverAppBar`** whose hero band holds the white OSTA logo (shrinks but stays visible as the body scrolls, back button pinned over it, subtle shadow once content scrolls under), then a bold centered title and optional subtitle — used by **all four** auth screens (`auth-choose`, login/register, forgot, reset) **plus the language pick and role chooser**, so the brand identity now carries across the whole logged-out flow (splash → language → role → onboarding → auth); previously only the chooser showed a logo, and language/role were flat centered columns. The language + role screens keep their selectable `_LanguageCard`/`_RoleCard` bodies (accent-border + check for the saved choice) but drop their ad-hoc logo/AppBar for the shared header. The chooser uses the full lockup (`AppImages.fullLogo`); the inner screens use the smaller wordmark (`AppImages.logo`) — both white assets that read on `AppColors.brandGreen` (the same green + white pairing the splash uses), and enlarged for presence (hero 180, inner wordmark 120). Login/register/forgot/reset form fields are now wrapped in an elevated `AppCard`. **Removed the hardcoded, non-theme-aware `socialButton`** (baked-in `Colors.black`/`white`/`grey.shade300`, radius 14, height 55 — unreadable in dark mode): both Google/Apple call sites now use the tokenized `AppButton` (secondary + icon) and `lib/features/onboarding/widget/social_button.dart` was deleted. Raw `TextStyle(color: error)` form errors (repeated in 3 files) were replaced by a themed **`AuthFormError`** widget. No new strings, dependencies, or routes; the email-sent / reset-success sub-views stay icon-led. `flutter analyze` clean; all 107 tests pass (incl. the boot-to-auth-choose widget test).

  > ‏**إعادة تصميم واجهة/تجربة شاشات المصادقة واللغة والدور + شعار موحّد على كل شاشة** (2026-07-08) — أُضيف `BrandScaffold` مشترك (`lib/shared/ui/brand_scaffold.dart`) — شريط علوي أخضر متقلّص (`SliverAppBar`) يحمل شعار OSTA الأبيض (يصغر لكن يبقى ظاهرًا عند التمرير، وزر الرجوع مثبّت فوقه)، ثم عنوان عريض متوسّط وعنوان فرعي اختياري — تستخدمه **الشاشات الأربع** (auth-choose، الدخول/التسجيل، نسيت كلمة المرور، إعادة التعيين) **إضافةً إلى شاشتَي اختيار اللغة والدور**، فأصبحت هوية العلامة ممتدّة عبر تدفّق غير المسجّل كله (كان الشعار على شاشة الاختيار فقط، وكانت شاشتا اللغة والدور أعمدة مسطّحة). تستخدم شاشة الاختيار الشعار الكامل (`AppImages.fullLogo`)، وتستخدم الشاشات الداخلية النسخة الأصغر (`AppImages.logo`) — وكلاهما أبيض يظهر جيدًا على `AppColors.brandGreen`. وأصبحت حقول النماذج داخل `AppCard` بارز. وأُزيل `socialButton` غير المتوافق مع الثيم (ألوان ثابتة، مكسور في الوضع الداكن) لصالح `AppButton` (ثانوي + أيقونة) وحُذف ملفه. واستُبدلت نصوص الأخطاء الخام بمكوّن `AuthFormError` موحّد. بلا نصوص أو حزم أو مسارات جديدة؛ وتبقى شاشات النجاح بالأيقونة. التحليل نظيف وكل الاختبارات (107) تنجح.
- **Auth-flow reorder + language/role/onboarding every logged-out launch + back chain** (2026-07-08) — reordered the logged-out flow to **`splash → language → role → onboarding → auth-choose → auth → shell`** (role now precedes onboarding). **The language, role, and onboarding screens now all re-show on every logged-out launch** (not just first install / first pick): each is gated by an in-memory `SessionState` flag — `languageAcknowledged`, `roleAcknowledged`, `onboardingAcknowledged` — reset every cold `bootstrap`, so the persisted locale/role are the pre-selected **defaults** (the redesigned language + role cards mark the saved choice with an accent border + check) but the screens still repeat while logged out. `chooseLanguage`/`chooseRole` set their flags; the role gate also still forces the chooser whenever `activeRole` is null (fresh or after "switch role"); a held token skips all three. **Back chain finished**: the auth-choose back button returns to **onboarding** (new `SessionController.resetOnboarding()` clears the in-memory ack; role + language stay); login/register back → auth-choose. Onboarding also gains a **Skip** button (top-right, hidden on the last slide) that acknowledges the intro and jumps to auth-choose. Redirect-guard + controller + first-run widget tests updated for the new order and gates. Note: the "language not showing" report was not a bug — the screen was correctly gated by the previously-saved locale; this change makes it (and the role chooser) repeat for logged-out users by request.
- **Auth-flow enhancements — onboarding gate + redesigned language & auth-choose screens** (2026-07-07) — the logged-out first-run flow is now `splash → language → onboarding → role → auth-choose → auth → shell`. **Onboarding** is reachable again and shows on **every launch while logged out**: a new in-memory `SessionState.onboardingAcknowledged` flag (reset each cold start via `bootstrap`/`clearingRole`) gates it in `resolveRedirect`, and its finish button calls `SessionController.acknowledgeOnboarding()` instead of navigating manually. The carousel is now intro-only (its old inline language/social blocks moved to dedicated screens) and fully localized. **`LanguagePage`** was redesigned (logo, subtitle, card options). A new **`AuthChoosePage`** (`/auth/choose`, the default unauthenticated landing) offers **Sign in / Create account** — routing into the shared `AuthPage` via a new `?mode=login|register` query param (read in `AuthPage`, applied through `AuthCubit.setMode`) — plus stubbed **Google/Apple** social buttons (a "coming soon" snackbar). Guard `authSurface` now whitelists `/auth/choose`. **Back navigation** (the flow is `context.go`-based, so there's no native pop stack): auth-choose's back button clears the role via `SessionController.switchRole()` → role chooser, and the login/register `AppBar` back button returns to `/auth/choose`; `clearingRole` now preserves `onboardingAcknowledged` so clearing the role doesn't bounce back into onboarding. EN + AR strings added; redirect guard + first-run widget tests updated. `mechanic`/`tow` and the no-guest-path model are unchanged.
- **Adopted a `develop`/`main` branching model** (2026-07-06) — introduced a long-lived **`develop`** integration branch: all feature/chore branches are now cut from `develop` and PR into `develop`, while **`main`** becomes the protected release branch, advanced **only** by a `develop → main` release PR + SemVer tag (`v0.<n>.0` per milestone, `v1.0.0` = MVP). Retargeted the open first-run PR ([#67](https://github.com/YoussefSalem582/Osta-App/pull/67)) to `develop`. Updated every workflow doc and AI-agent config (`AGENTS.md`, `CONTRIBUTING.md`, `README.md`, `ARCHITECTURE.md`, `OSTA_plan.md` §13, `OSTA_TODO.md`, `CURRENT_STATUS.md`, `guides/01_folder_structure.md` + `guides/03_how_to_add_new_feature.md`, `reference/COMMON_PITFALLS.md`, `.cursor/rules/git-commits.mdc`, `.claude/commands/{add-feature,new-screen}.md`, and the `.agents`/`.cursor` `add-feature` skills) and added `develop` to the CI `push` triggers (`.github/workflows/ci.yml`).

- **Branch-naming rule tightened across all docs and AI-agent configs** (2026-07-05) — branch names must be hand-written, descriptive, lowercase kebab-case (`<type>/<issue>-<slug>`, e.g. `feat/44-booking-funnel`, `fix/auth-401-loop`); auto-generated/tool-default names (random suffixes, `claude/...`, `cursor/...`, `codex/...`) are forbidden and must be renamed (`git branch -m`) before opening a PR. Updated: `AGENTS.md` (naming table + Git & PRs), `CONTRIBUTING.md`, `README.md`, `ARCHITECTURE.md`, `OSTA_plan.md` (§2 NEVER list + §13.1), `OSTA_TODO.md` (per-epic ritual), `osta_readme_files/guides/03_how_to_add_new_feature.md`, `osta_readme_files/CURRENT_STATUS.md`, `.cursor/rules/git-commits.mdc`, `.claude/commands/add-feature.md` + `new-screen.md`, and the `add-feature` skills in `.agents/` and `.cursor/`.

- **Docs synced to the plain-Dart refactor + made bilingual (EN/AR)** (2026-07-05) — after the "defer advanced tooling" refactor ([PR #69](https://github.com/YoussefSalem582/Osta-App/pull/69)) dropped `fpdart`, `freezed`, `json_serializable`, `injectable`, `build_runner`, build flavors, the `/gallery` route, and the 3-job CI matrix, the whole documentation set was corrected to match (sealed `Failure` + `try`/`catch`; plain `Equatable` models + hand-written JSON; manual `get_it` DI; single `BASE_URL`; one `format · analyze · test` CI job) and cross-linked to [`docs/ROADMAP.md`](docs/ROADMAP.md). ### Added

- **M1 Auth — email/password surfaces + secure storage + password recovery** (epic [#35](https://github.com/YoussefSalem582/Osta-App/issues/35)) (2026-07-06) — extended the shared auth surface toward the M1 email+password epic (no phone OTP): the **register** form now collects a unique username and a required Egyptian **+20** phone (masked prefix, normalized to E.164 on submit), confirms the password, and gates submit behind a **Terms & Privacy** checkbox; the **login** form gains a password **visibility toggle** and a **"forgot password?"** link. Added a two-step **password-recovery flow** — `ForgotPasswordPage` (`POST /forgot-password`) → `ResetPasswordPage` (`POST /reset-password`, email/token prefilled from a reset-link's query params) — driven by a new `PasswordRecoveryCubit`, on new `/auth/forgot-password` + `/auth/reset-password` routes whitelisted in the `resolveRedirect` guard. `AuthRepository` gained `logout` (`POST /auth/logout`, best-effort revoke that always clears the token pair — wired through `SessionController.signOut`), `forgotPassword`, and `resetPassword`. Server **422** responses now surface as inline per-field errors (email/username/phone/password) via `AuthState.fieldErrors`, and validators moved to a shared `AuthValidators` helper. EN + AR strings for the whole surface; endpoints follow the canonical catalogue (`/auth/logout`, public `/forgot-password` + `/reset-password`). Tests cover the new repo calls, the recovery cubit, the validators, and the password toggle.
- **Debug-only login prefill** (2026-07-07) — `AuthPage` prefills the email/password fields with the App Review / QA test account (`test@osta.com` / `osta123123`) under `kDebugMode` only; release builds compile the block out and ship empty fields. Speeds local sign-in and gives App Review a one-tap login. The account must exist backend-side — the app still authenticates against `/auth/login`.
- **First-run flow & 4-role split** (epic [#32](https://github.com/YoussefSalem582/Osta-App/issues/32), [PR #67](https://github.com/YoussefSalem582/Osta-App/pull/67)) (2026-07-06) — OSTA's "one download, then split" first-run model: `SplashPage` bootstraps persisted `{token, activeRole, locale}`; a one-time `LanguagePage` (ar/en, RTL-first); a `RoleChooserPage` where `customer` + `business` are live and `mechanic` + `tow` render disabled ("coming soon"); a pure `resolveRedirect` guard driving a single `go_router` global redirect (splash → language → chooser → auth → shell, no guest path); `account_type = activeRole` on register/login; wrong-shell self-heal from `me.type` + toast; "switch role" (clears role, keeps token) and sign-out; customer + business shells. Router state lives in a `SessionController` cubit (the routing source of truth). **Rebased onto current `main` and adapted to the post-[#69](https://github.com/YoussefSalem582/Osta-App/pull/69) plain-Dart conventions** — the feature's `injectable`/`freezed`/`fpdart` and the dev `/gallery` route were dropped: session/auth deps are now registered by hand in `configureDependencies()` (`AuthCubit` as a factory) and the `AppRouter` is built with the `SessionController` singleton.
